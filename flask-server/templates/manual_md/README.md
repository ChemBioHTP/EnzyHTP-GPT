Manual MD Input Package
=======================

This package contains MD input files generated by EnzyHTP plus a small CLI tool
to help submit and manage high-throughput MD jobs on your cluster.

Package Layout
--------------
./
  README.md
  manifest.json
  md_cli.py
  md_cli_config.json
  inputs/
    <mutant_name>/
      MD/
        rep_0/
        rep_1/
        ...
  jobs/
    <job_id>.sh
  generated_scripts/

Key Files
---------
- manifest.json: job metadata (commands, work_dir, replica, mutant).
- inputs/: EnzyHTP-generated MD input files.
- jobs/: default job scripts generated by EnzyHTP (Slurm/ACCRE defaults).
- md_cli.py: CLI to generate scripts and submit/manage jobs.
- md_cli_config.json: scheduler config and script templates.

Quick Start (Slurm example)
---------------------------
1) Review the inputs:
   - Check manifest.json for mutants/replicas.
   - Inspect inputs/<mutant>/MD/ for generated MD files.

2) (Optional) Customize the scheduler template:
   - Edit md_cli_config.json
   - Update submit/status/cancel commands and script_template.

3) Generate scripts and submit:
   python md_cli.py submit --package . --scheduler slurm

4) Check status:
   python md_cli.py status --package . --scheduler slurm

5) Cancel jobs:
   python md_cli.py cancel --package . --scheduler slurm

Notes
-----
- Constraints: if only atom IDs were provided, target values are set to the
  current geometry in the starting structure.
- The default job scripts are generated with ACCRE/Slurm settings; most users
  should adjust resource lines in md_cli_config.json.
